# LiteLLM NVIDIA NIM Proxy

A lightweight local proxy that aggregates multiple NVIDIA NIM API keys for automatic rotation, built on [LiteLLM](https://github.com/BerriAI/litellm).

## Features

- **Automatic key rotation** - Multiple API keys rotate automatically per model
- **119+ models** - All NVIDIA NIM models available out of the box
- **Usage-based routing** - Prioritizes keys with remaining quota
- **Unified API** - Single endpoint for all models, compatible with OpenAI format

## Prerequisites

- Python (tested with 3.10+)
- NVIDIA NIM API key(s) from [build.nvidia.com](https://build.nvidia.com)
- Git

## Quick Start

### 1. Clone and Install

`ash
git clone <your-repo-url>
cd LLM-API-Key-Proxy
F:\miniconda3\Scripts\pip install litellm pyyaml
`

Replace F:\miniconda3\ with your Python path.

### 2. Configure API Keys

Copy the example env file and fill in your keys:

`ash
copy .env_litellm.example .env_litellm
`

Edit .env_litellm:

`env
NVIDIA_NIM_API_KEY_1=nvapi-your-first-key
NVIDIA_NIM_API_KEY_2=nvapi-your-second-key
LITELLM_MASTER_KEY=sk-your-random-key
`

### 3. Update Model List

`ash
update_models.bat
`

This fetches the latest available models from NVIDIA NIM and generates config_litellm.yaml.

### 4. Start the Proxy

`ash
start_litellm.bat
`

The server starts on http://127.0.0.1:5000. Press Ctrl+C to stop.

### 5. Verify

`ash
curl http://127.0.0.1:5000/v1/models -H "Authorization: Bearer <LITELLM_MASTER_KEY>"
`

## API Usage

### Chat Completions

`ash
curl http://127.0.0.1:5000/v1/chat/completions \
  -H "Authorization: Bearer <LITELLM_MASTER_KEY>" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "nvidia_nim/meta_llama-3.1-8b-instruct",
    "messages": [{"role": "user", "content": "Hello"}]
  }'
`

### Available Models

After starting, browse http://127.0.0.1:5000/ or call:

`ash
curl http://127.0.0.1:5000/v1/models -H "Authorization: Bearer <LITELLM_MASTER_KEY>"
`

## Changing the Port

If port 5000 is in use, edit start_litellm.bat and config.toml:

`at
:: In start_litellm.bat, change:
--port 5000
--host 127.0.0.1
`

`	oml
# In config.toml
base_url = "http://127.0.0.1:5000"  # change this
`

## How Key Rotation Works

Each model in config_litellm.yaml is configured with multiple deployments (one per API key). LiteLLM automatically load-balances across them using usage-based-routing, prioritizing keys with remaining quota. If one key is exhausted or fails, it automatically switches to the next.

## Notes

- API keys are stored in .env_litellm - never commit this file
- config_litellm.yaml is auto-generated by update_models.py - do not edit manually
- Re-run update_models.bat periodically to sync with NVIDIA NIM model updates
