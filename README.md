# LiteLLM NVIDIA NIM Proxy

本地代理服务，聚合多个 NVIDIA NIM API Key 实现自动轮换，基于 [LiteLLM](https://github.com/BerriAI/litellm) 构建。

Local proxy that aggregates multiple NVIDIA NIM API keys for automatic rotation, built on [LiteLLM](https://github.com/BerriAI/litellm).

---

## Directory Structure / 目录结构

`
LLM-API-Key-Proxy/
├── config_litellm.yaml    ← 自动生成（regenerated by update_models.py，勿手动编辑）
├── .env_litellm           ← 你的真实 API Key（NEVER commit 此文件）
│                          ← Your real API keys (NEVER commit this file)
├── .env_litellm.example  ← API Key 模板（复制一份填入你的 key）
│                          ← Template: copy and fill in your keys
├── .gitignore             ← 忽略 .env_litellm 和 config_litellm.yaml
│                          ← Ignores .env_litellm and config_litellm.yaml
├── README.md              ← 本文件 / This file
│
├── start_litellm.bat      ← 启动代理服务 / Start the proxy server
├── stop_litellm.bat       ← 关闭服务并释放端口 / Stop server and release port
├── update_models.bat      ← 从 NVIDIA NIM 获取最新模型列表 / Fetch latest models
├── free_port.ps1          ← 强制释放端口（端口卡住时用）/ Force release port if stuck
│
├── update_models.py       ← 更新脚本核心（被 update_models.bat 调用）
│                          ← Core script called by update_models.bat
│
└── config.toml            ← Codex CLI 配置文件 / Codex CLI config file
`

## Installation / 安装

### 1. 安装 Python 依赖 / Install Python dependencies

`ash
# 将 D:\miniconda3 替换为你的 Python 路径
# Replace D:\miniconda3 with your Python path
D:\miniconda3\Scripts\pip install litellm pyyaml
`

### 2. 配置环境变量 / Configure environment variables

`ash
copy .env_litellm.example .env_litellm
`

编辑 .env_litellm，填入你的 key：

`env
NVIDIA_NIM_API_KEY_1=nvapi-your-first-key
NVIDIA_NIM_API_KEY_2=nvapi-your-second-key
LITELLM_MASTER_KEY=sk-your-random-key
`

> **如何生成 LITELLM_MASTER_KEY？**
> 运行：python -c "import secrets; print('sk-' + secrets.token_hex(16))"

### 3. 更新模型列表 / Update model list

`ash
update_models.bat
`

### 4. 启动 / Start

`ash
start_litellm.bat
`

服务运行在 http://127.0.0.1:5000，按 Ctrl+C 停止。

## How It Works / 工作原理

### 调用链 / Call Chain

`
start_litellm.bat
  → litellm.exe (D:\miniconda3\Scripts\litellm.exe)
    → Python (D:\miniconda3\python.exe)
      → litellm 库 (D:\miniconda3\Lib\site-packages\litellm\)
        → 加载 config_litellm.yaml
          → 启动 uvicorn HTTP 服务
            → 监听 127.0.0.1:5000
`

litellm.exe 是 pip 安装时生成的入口脚本，找到同目录的 Python 后运行 litellm 库代码。

litellm.exe is a shim created by pip. It finds the Python interpreter in the same directory and runs the litellm library code.

### Key 轮换 / Key Rotation

`
config_litellm.yaml 中每个模型 N 个 deployment：
nvidia_nim/01-ai_yi-large  → NVIDIA_NIM_API_KEY_1
nvidia_nim/01-ai_yi-large  → NVIDIA_NIM_API_KEY_2
nvidia_nim/deepseek-v4-pro → NVIDIA_NIM_API_KEY_1
nvidia_nim/deepseek-v4-pro → NVIDIA_NIM_API_KEY_2
...

LiteLLM router 自动 round-robin + usage-based-routing
→ 优先用配额剩余的 key，失败自动切换下一个
→ Prioritizes keys with remaining quota, auto-failover on error
`

### API 调用示例 / API Usage Example

`ash
curl http://127.0.0.1:5000/v1/chat/completions \
  -H "Authorization: Bearer <LITELLM_MASTER_KEY>" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "nvidia_nim/meta_llama-3.1-8b-instruct",
    "messages": [{"role": "user", "content": "Hello"}]
  }'
`

## Change Port / 修改端口

如果 5000 端口被占用，修改以下两处：

1. start_litellm.bat 中的 --port 5000
2. config.toml 中的 ase_url = "http://127.0.0.1:5000"
